Project Summary

With the help of Professor Timothy Takahashi, I've been able to establish a research project with Arizona State's Fulton Undergraduate Research Initiative (FURI). As part of FURI, I have developed a flight controller and associated software capable of logging data regarding the flight of an aircraft. Using a combination of in-flight filtration and post-flight analysis, the objective of this project is to determine and optimize certain Aerodynamic features, mainly regarding the Thrust lapse of the aircraft.

The flight controller (fig 2) used to collect and log sensor data consists of the following sensors:

  1. IMU - 6 axis
  2. Barometer
  3. Pitot Tube
  4. Thrust Sensor (Load Cell)


The current setup collects data at a sampling rate of 100Hz (every 10ms) which gives users plenty of data to work with. All data collected is also logged onto a micro sd card which is analyzed post-flight.

Figures 3 and 4 are snippets of the two layers of code running. Figure 3 runs on the arduino to collect and log sensor data to the sd card. Figure 4 runs post flight on the SD card and simulates a digital and analog filter. This enables noisy analog data collected to be usable.

Although my research period with FURI is complete, I plan to continue enhancing this project to eventually include a virtual reconstruction of the aircraft's flight for further analysis considering thrust lapse data can be skewed by undersired aircraft manuvers.
